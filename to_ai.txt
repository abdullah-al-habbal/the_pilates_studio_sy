    public function verify(VerifyOtpRequest $request): JsonResponse
    {
        /** @var User $user */
        $user = User::where('email', $request->email)->firstOrFail();
        if (! is_null($user->email_verified_at)) {
            return $this->error('Email is already verified.', 422);

            'otp_code' => null,
            'otp_expires_at' => null,
        ]);
        $token = $user->createToken('mobile')->plainTextToken;
        return $this->success([
            'token' => $token,
            'user' => new UserResource($user),

{
    public function __construct(
        private readonly AuthService $authService
    ) {}
    #[Endpoint('Register', description: 'Register a new user.')]
    public function register(RegisterRequest $request): JsonResponse
    {
        $user = $this->authService->register($request->validated());
        return $this->created(
            ['email' => $user->email],
            'Registration successful. Please verify your email.',
        );
    }
    #[Endpoint('Login', description: 'Authenticate a user and return a token.')]
    public function login(LoginRequest $request): JsonResponse
    {
        $data = $request->validated();
        $user = $this->authService->attemptLogin($data['email'], $data['password']);
        if (! $user) {

        }
        if (! $user->isActive()) {
            return $this->error('Your account has been deactivated.', 403);
        }
        if (is_null($user->email_verified_at)) {
            $this->authService->sendOtp($user);
            return $this->error(
                'Email not verified. A new OTP has been sent to your email.',
                403,
                ['email_verified' => false],
            );
        }
        $deviceName = $data['device_name'] ?? null;
        $token = $this->authService->createToken($user, $deviceName);

            'token' => $token,
            'user' => new UserResource($user),
        ], 'Login successful.');
    }

{
    public function __construct(
        private readonly UserService $userService
    ) {}
    public function register(array $data): User
    {
        $user = $this->userService->createUser($data);
        $this->sendOtp($user);
        return $user;
    }
    public function attemptLogin(string $email, string $password): ?User
    {
        $user = $this->userService->findByEmail($email);
        if (! $user || ! $this->userService->validateCredentials($user, $password)) {
            return null;
        }
        return $user;
    }
    public function createToken(User $user, ?string $deviceName = null): string
    {
        $tokenName = $deviceName ?? 'auth_token_'.Carbon::now()->timestamp;
        return $user->createToken($tokenName)->plainTextToken;
    }
    public function sendOtp(User $user): void
    {
        $otp = $this->userService->generateOtp();
        $this->userService->saveOtp($user, $otp);
    }
    public function logout(User $user): void
    {
        $user->currentAccessToken()?->delete();
    }
}

use Illuminate\Support\Facades\RateLimiter;
use Illuminate\Support\Facades\URL;
use Illuminate\Support\ServiceProvider;
use Illuminate\Validation\Rules\Password;
class ApplicationServiceProvider extends ServiceProvider
{
    public function register(): void {}
    public function boot(): void
    {
        RateLimiter::for('api', function (Request $request) {
            return Limit::perMinute(60)->by($request->user()?->id ?: $request->ip());
        });
        $this->configureModelProtections();
        $this->configureDatabase();
        $this->configurePasswordDefaults();
    }
    protected function configureModelProtections(): void
    {
        $isLocal = $this->app->environment('local', 'development');
        Model::preventLazyLoading($isLocal);
        Model::preventSilentlyDiscardingAttributes($isLocal);
        Model::preventAccessingMissingAttributes($isLocal);
        Model::shouldBeStrict($isLocal);
        Model::unguard(false);
    }
    protected function configureDatabase(): void
    {
        DB::prohibitDestructiveCommands($this->app->isProduction());
        if ($this->app->environment('production')) {
            if (config('app.force_https', false)) {
                URL::forceScheme('https');
            }
            DB::statement('SET SESSION sql_mode = STRICT_ALL_TABLES');
            DB::statement('SET SESSION group_concat_max_len = 1000000');
        } else {
            DB::listen(function ($query) {
                if ($query->time > 500) {
                    logger()->warning('Slow query detected', [
                        'sql' => $query->sql,
                        'bindings' => $query->bindings,
                        'time' => $query->time,
                        'connection' => $query->connectionName,
                    ]);
                }
            });
        }
    }
    protected function configurePasswordDefaults(): void
    {
        Password::defaults(function () {
            return Password::min(8)
                ->mixedCase()
                ->letters()
                ->numbers()
                ->symbols()
                ->uncompromised();
        });
    }

Architectural Review
What you're doing well:

Strict declare(strict_types=1) everywhere
Clean DI via constructor promotion with readonly
Proper layering: Controller → Service → Repository
Feature-based namespace organization
Scramble annotation consistency
BaseApiController abstraction for response formatting

Issues identified:
UserEloquentRepository — UserRegisteredEvent::dispatch() is fired inside the repository. That's a domain concern leaking into the persistence layer. Events belong in the Service, not the repo. Also, createInTransaction wraps create() which already has its own transaction — double-wrapping.
BookingEloquentRepository — listUserBookings and findByUser are wrapped in DB::transaction(). Read operations must never be in transactions. Transactions are for writes only.
BookingService — adjustCredits, expireBooking, refundBooking each fire a transaction for a single update() call. No transaction needed for a single atomic write. Also, decrementCredits wraps a repo call in a transaction at the service layer — redundant if the repo handles it. lockForUpdate is a separate method but is also baked into adjustCredits directly with Booking::lockForUpdate() — bypasses the repository entirely, breaking SoC.
ClassSessionService — find, lockForUpdate, hasAvailableSpots are three methods that can be collapsed as you suggested. Good instinct.
LoggingService — Don't inject it into repositories. Logging is an application/service concern, not a persistence concern. Repositories should be thin data-access objects only.
EmailVerificationController — Business logic (OTP validation, hashing, $user->update(...), token creation) lives directly in the controller. This must move to AuthService/UserService.
ProfileController — $user->tokens()->delete() and $user->update(['deleted_by' => ...]) are business logic in the controller. Must move to a service.
SettingController — firstOrCreate with business logic inline. Should be in a SettingService.
